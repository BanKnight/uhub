## Learnings
- Better Auth 的真实返回对象包含 `handler(request: Request)`，适合直接挂到 Hono 路由
- Drizzle D1 的真实导出路径是 `drizzle-orm/d1`
- Better Auth 的 Drizzle 适配器签名是 `drizzleAdapter(db, { provider: 'sqlite', schema })`
- 现阶段先做“最小可编译接线”是对的；后面再补 admin 登录页与 auth client 的细节实现
- Wrangler 4.81.1 的 D1 migration 路径配置不是顶层 `migrations_dir`，而是写在 `d1_databases[]` 项内
- Better Auth 在 Cloudflare Worker 本地 dev 下需要 `nodejs_compat`，否则会出现 AsyncLocalStorage 能力告警
- tRPC 裸 HTTP 调用时，当前这套路由的 input body 直接传业务对象即可，不需要额外包一层 `json`
- API key 的“只能给 active channel 签发”必须落在后端服务层，而不应只依赖前端筛选

## Learnings
- 当前并发门禁最合适的插点是 `requireApiKey` 之后、真实上游 `fetch` 之前，这样不会污染 key 校验逻辑，也便于后续给 `/anthropic` 复用
- 本地 Cloudflare Durable Object 在当前项目里使用“普通导出类 + (state, env) 构造签名”即可，不需要继承可实例化基类
- `429` 是在 acquire 失败前返回的，因此当前实现不会为被拒绝的并发请求写入 `requests` 记录；如果后续要做审计，再单独补 rejected request 记录即可
